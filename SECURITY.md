# Segurança

> [!IMPORTANT]
> Política completa em `docs/CODEX_RUNBOOK.md`. Para vulnerabilidades, envie email para `security@zappro.site`. Não abra issues públicas.

- Nunca commitar segredos (tokens, senhas, chaves). Use variáveis de ambiente e `.env.example` apenas como referência.
- Pasta `secrets/` proibida para agentes; não versionar dados sensíveis.
- Relate vulnerabilidades de forma privada (veja `docs/CODEX_RUNBOOK.md`).
